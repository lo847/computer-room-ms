# 机房人员信息管理系统 软件开发设计方案

## 一、项目名称

机房人员信息管理系统（Computer Room User Management System）

---

## 二、项目背景

在学校机房、实验室、工作室等场景中，成员信息管理是一项高频但低效的工作。目前多数机房仍依赖 Excel 表格或纸质登记，存在查询不便、信息更新滞后、历史记录难以追溯等问题，管理成本高且容易出错。

为此，开发一套基于 B/S 架构的机房人员信息管理系统，实现成员注册、登录认证、个人信息维护、头像上传、成员查询与统计等功能，将机房人员管理全面数字化，提升管理效率与数据可靠性。

---

## 三、项目目标

开发一个基于浏览器/服务器架构的信息管理平台，实现以下核心能力：

1. 用户自助注册与登录
2. 个人信息录入与维护
3. 头像上传与管理
4. 全部成员信息浏览
5. 按条件搜索成员
6. 成员详细资料查看
7. 管理员对用户的增删改查
8. 数据统计与可视化

---

## 四、系统角色设计

### 4.1 普通用户

- 注册账号
- 登录系统
- 修改个人资料（姓名、手机号、邮箱、毕业届数、头像）
- 修改密码
- 浏览全部成员信息
- 查看成员详细资料
- 按姓名/学号/毕业届数搜索成员

### 4.2 管理员

管理员拥有普通用户全部权限，额外具备：

- 查看全部用户列表
- 修改任意用户信息
- 删除用户
- 重置用户密码
- 禁用/启用用户账号
- 查看数据统计（总人数、各届人数分布、近期注册趋势、在线人数）

## 五、功能模块设计

### 5.1 用户注册模块

新用户首次使用时通过注册页面创建账号。

**注册字段：**

| 字段 | 类型 | 必填 |
| --- | --- | --- |
| 学号 | varchar(20) | 是 |
| 姓名 | varchar(50) | 是 |
| 密码 | varchar(100) | 是 |
| 确认密码 | varchar(100) | 是 |
| 性别 | varchar(10) | 否 |
| 手机号 | varchar(20) | 否 |
| 邮箱 | varchar(100) | 否 |
| 毕业届数 | varchar(20) | 是 |
| 头像 | varchar(255) | 否 |

**注册流程：**

用户填写表单 → 前端校验字段格式 → 后端校验学号唯一性 → 密码加密存储 → 写入数据库 → 返回注册成功

**校验规则：**

- 学号不可与已有记录重复
- 密码长度不少于 6 位
- 两次输入的密码必须一致
- 手机号、邮箱若填写则校验格式

---

### 5.2 用户登录模块

用户通过学号与密码登录系统。

**登录流程：**

输入学号 + 密码 → 验证码校验 → 数据库比对 → 生成 Token（JWT） → 登录成功 → 跳转首页

**安全措施：**

- 密码采用 BCrypt 加密存储，永不明文落库
- 登录时需通过图形验证码，防止暴力破解
- 连续登录失败 5 次后锁定账号 15 分钟
- 登录成功后签发 JWT，后续请求携带 Token 鉴权

---

### 5.3 个人信息管理模块

登录用户可在"个人中心"页面维护自己的信息。

**可修改字段：**

- 姓名
- 手机号
- 邮箱
- 头像
- 毕业届数

**密码修改：**

需输入原密码验证身份后再设置新密码。

---

### 5.4 头像上传模块

用户可在注册时或个人信息页上传头像。

| 项目 | 说明 |
| --- | --- |
| 支持格式 | JPG、PNG、JPEG、GIF、WebP |
| 大小限制 | 不超过 5 MB |
| 存储方式 | 上传至服务器 `/uploads/avatar/` 目录，数据库保存相对路径 |
| 默认头像 | 未上传时显示姓名首字占位图 |

---

### 5.5 成员信息展示模块

首页以卡片形式展示全部成员，每张卡片包含：

- 头像（或首字占位图）
- 姓名
- 学号
- 毕业届数

卡片支持点击查看详情，支持删除操作（管理员权限）。

---

### 5.6 成员搜索模块

搜索栏支持以下筛选条件：

- 按姓名模糊搜索
- 按学号精确搜索
- 按毕业届数筛选

多个条件可组合使用，搜索结果实时更新。

---

### 5.7 成员详情模块

点击成员卡片进入详情页，展示完整信息：

- 头像
- 姓名
- 学号
- 毕业届数
- 手机号
- 邮箱
- 性别
- 注册时间
- 最后登录时间

---

### 5.8 管理员模块

管理员拥有独立的用户管理后台。

**用户管理：**

- 用户列表（分页展示，支持搜索与筛选）
- 编辑用户信息
- 删除用户（软删除或物理删除）
- 禁用/启用用户账号
- 重置用户密码

**数据统计：**

- 系统总人数
- 各毕业届数人数分布（表格 + 图表）
- 最近 30 天注册人数趋势
- 当前在线人数

## 六、数据库设计

### 6.1 用户表 `user`

```sql
CREATE TABLE user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    student_no      VARCHAR(20)  NOT NULL UNIQUE COMMENT '学号',
    name            VARCHAR(50)  NOT NULL COMMENT '姓名',
    password        VARCHAR(255) NOT NULL COMMENT '密码（BCrypt 加密）',
    gender          VARCHAR(10)  DEFAULT NULL COMMENT '性别',
    phone           VARCHAR(20)  DEFAULT NULL COMMENT '手机号',
    email           VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    avatar          VARCHAR(255) DEFAULT NULL COMMENT '头像路径',
    graduation_year VARCHAR(20)  NOT NULL COMMENT '毕业届数',
    role            VARCHAR(20)  NOT NULL DEFAULT 'user' COMMENT '角色: user / admin',
    status          TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 1-正常 0-禁用',
    create_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    update_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 6.2 登录日志表 `login_log`

```sql
CREATE TABLE login_log (
    id         BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    user_id    BIGINT   NOT NULL COMMENT '用户 ID',
    login_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '登录时间',
    login_ip   VARCHAR(50) DEFAULT NULL COMMENT '登录 IP',
    FOREIGN KEY (user_id) REFERENCES user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='登录日志表';
```

## 七、系统架构设计

采用前后端分离 + MVC 三层架构：

```
┌─────────────────────────┐
│   前端 (Vue 3 + Element Plus)  │
└───────────┬─────────────┘
            │ HTTP / JSON
            ▼
┌─────────────────────────┐
│   Controller 控制层          │
│   接收请求、参数校验、返回响应    │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│   Service 业务层            │
│   业务逻辑、事务管理、权限校验    │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│   Mapper 数据访问层          │
│   MyBatis / JPA 持久化      │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│   MySQL 8.0 数据库          │
└─────────────────────────┘
```

**补充组件：**

- **Redis**：缓存用户 Session 与热点数据，降低数据库压力
- **JWT**：无状态身份认证，Token 在 Redis 中管理有效期与刷新
- **Nginx**：前端静态资源托管 + 反向代理

## 八、技术选型

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端框架 | Vue 3 | 渐进式前端框架，组件化开发 |
| UI 组件库 | Element Plus | 成熟的企业级 Vue 3 组件库 |
| 前端构建 | Vite | 快速开发与构建工具 |
| 后端框架 | Spring Boot 3.x | Java 主流微服务框架 |
| ORM 框架 | MyBatis-Plus | 简化 CRUD，提升开发效率 |
| 数据库 | MySQL 8.0 | 关系型数据库，存储核心业务数据 |
| 缓存 | Redis 7.x | 加速热点数据访问，管理登录状态 |
| 认证 | JWT + Spring Security | 无状态 Token 认证 |
| 文件存储 | 本地磁盘 | 头像等静态文件 |
| 反向代理 | Nginx | 生产环境部署 |

## 九、API 接口设计

| 方法 | 路径 | 说明 | 权限 |
| --- | --- | --- | --- |
| POST | `/api/auth/register` | 用户注册 | 公开 |
| POST | `/api/auth/login` | 用户登录 | 公开 |
| GET | `/api/auth/captcha` | 获取验证码 | 公开 |
| GET | `/api/users/me` | 获取当前用户信息 | 登录 |
| PUT | `/api/users/me` | 修改个人资料 | 登录 |
| PUT | `/api/users/me/password` | 修改密码 | 登录 |
| POST | `/api/users/me/avatar` | 上传头像 | 登录 |
| GET | `/api/users` | 成员列表（分页+搜索） | 登录 |
| GET | `/api/users/:id` | 成员详情 | 登录 |
| PUT | `/api/users/:id` | 修改用户信息 | 管理员 |
| DELETE | `/api/users/:id` | 删除用户 | 管理员 |
| PUT | `/api/users/:id/status` | 禁用/启用用户 | 管理员 |
| POST | `/api/users/:id/reset-password` | 重置密码 | 管理员 |
| GET | `/api/admin/stats` | 数据统计 | 管理员 |

## 十二、后期扩展方向

学生进入机房时扫码或刷卡签到，系统记录进出时间，生成考勤报表。

集成摄像头实现人脸识别身份验证，替代传统账号密码登录。

### 12.3 在线通知与公告

管理员可发布机房通知、维护公告、活动信息，成员在首页即可查看。

### 12.4 成员风采展示

展示优秀毕业生、竞赛获奖成员等信息，丰富系统内容。

### 12.5 设备借用管理

登记机房设备借用与归还记录，追踪设备流转状态。

## 十三、项目总结

本系统以机房成员管理为核心，完整覆盖用户注册、登录认证、信息录入、头像上传、成员查询搜索、管理员后台等全链路功能。系统采用前后端分离架构，后端基于 Spring Boot + MyBatis-Plus，前端采用 Vue 3 + Element Plus，以 MySQL 存储业务数据，Redis 加速热点访问，JWT 实现无状态认证，整体架构清晰、层次分明，具备良好的扩展性与可维护性。

本项目适合作为 Java Web 课程设计、数据库课程设计、软件工程课程设计以及毕业设计选题，能够切实满足机房成员信息管理与展示的实际需求。
